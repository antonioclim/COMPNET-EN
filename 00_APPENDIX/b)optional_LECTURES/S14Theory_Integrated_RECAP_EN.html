📦 Encapsulation

Concept Application → [User Data] Transport → [TCP/UDP Hdr | Data] = Segment Network → [IP Hdr | Segment] = Packet Data Link → [Eth Hdr | Packet | FCS] = Frame

Predict: minimum overhead for 1 byte of TCP payload over Ethernet?

14 (Eth) + 20 (IP) + 20 (TCP) = 54 bytes overhead for 1 byte of data ≈ 98% overhead.

This is why Nagle's algorithm coalesces small writes.

🔍 ARP — Bridging L2 and L3

Address Resolution Protocol (RFC 826)

Maps IPv4 to MAC within a broadcast domain. Without ARP, IP packets cannot become Ethernet frames.

Request (broadcast): "Who has 10.0.0.2? Tell 10.0.0.1"

Reply (unicast): "10.0.0.2 is at aa:bb:cc:dd:ee:ff"

Verify: ip neigh or arp -a

⚠️ ARP Poisoning

ARP has no authentication. Any host can forge replies — basis of MITM attacks.

Transport Layer

TCP vs UDP, connection management and ports

⚔️ TCP vs UDP

PropertyTCPUDP
ConnectionYes (3-way handshake)No
ReliabilityGuaranteed (ACK + retransmission)Best-effort
OrderingGuaranteed (sequence numbers)No
Flow controlYes (sliding window)No
Header size20+ bytes8 bytes

Rule of thumb: TCP when correctness matters; UDP when speed matters and loss is tolerable. QUIC combines both.

🤝 TCP Handshake

🖥️ Client
→ SYN (seq=x) →
← SYN-ACK (seq=y, ack=x+1) ←
→ ACK (ack=y+1) →
🖥️ Server
📜 Panix SYN Flood (1996)

First major DDoS: spoofed SYN packets exhausted the connection table. Defence: SYN Cookies (RFC 4987) — state encoded in the sequence number.

👋 FIN vs RST

Graceful (FIN):

Client→FIN→Server
Client←ACK←Server
Client←FIN←Server
Client→ACK→Server

Both sides agree. Data delivered.

RST — Something Wrong
  • Port not listening
  • Firewall rejecting
  • App crashed
  • Invalid segment

Routing table contains:

  • Destination + mask
  • Next-hop
  • Interface
  • Metric

Longest Prefix Match

Most specific route wins:

  • 0.0.0.0/0 → /0
  • 10.0.0.0/8 → /8
  • 10.0.0.0/24 → chosen

🌐 BGP Hijacking

⚡ YouTube–Pakistan (2008)

Pakistan Telecom announced a more specific BGP prefix for YouTube's IPs. It leaked globally. For ~2 hours, YouTube was black-holed. BGP has no built-in verification.

Lesson

RPKI adds cryptographic validation to BGP. Adoption growing but incomplete.

Section 5

Diagnostic Methodology

From "it doesn't work" to evidence-based diagnosis

✅ Six-Step Checklist

Methodology 1. Reproduce └─ Same request, topology, version? 2. L3 Connectivity └─ ping, ip route get, ip neigh 3. Ports & Sockets └─ ss -lntp (listening?) └─ ss -tnp (active?) 4. Capture └─ tcpdump -i <iface> host <ip> -w cap.pcap 5. Interpret └─ Handshake? Retransmissions? RST? HTTP codes? 6. Fix & Validate └─ Small change → measure → compare

🔗 Symptoms → Causes

SymptomCauseEvidence

504 Gateway TimeoutBackend slow/crashedLB logs + backend pcap

🔬 Refused vs Timeout

Refused $ curl http://localhost:9999/ curl: (7) Connection refused # pcap: SYN → RST
Timeout $ curl --connect-timeout 3 \ http://localhost:8888/ curl: (28) Connection timed out # pcap: SYN → SYN → SYN
Key Difference

Refused = RST = port closed. Timeout = no reply = firewall/routing.

📡 PCAP Analysis

Bash $ sudo tcpdump -i eth0 host 10.0.14.100 \ -w capture.pcap $ tshark -r capture.pcap -q -z conv,tcp $ tshark -r capture.pcap -Y "http.request" \ -T fields -e ip.src -e http.request.uri $ tshark -r capture.pcap \ -Y "tcp.analysis.retransmission" | wc -l

🔓 Solving the Friday 502

Remember slide 2? Apply the checklist:

Diagnosis 1. Reproduce: curl → 502 confirmed 2. L3: ping backends → OK 3. Ports: ss on backends → 8000 NOT listening! └─ Deploy changed port to 8080 4. Capture: SYN to :8000 → RST 5. Root cause: config mismatch 6. Fix: LB config → :8080 → done

4 commands, under 3 minutes.

Practical Patterns

Socket programming, load balancing, containers

🔄 TCP Socket Patterns

Python — Server import socket srv = socket.socket( socket.AF_INET, socket.SOCK_STREAM ) srv.setsockopt( socket.SOL_SOCKET, socket.SO_REUSEADDR, 1 ) srv.bind(("0.0.0.0", 9090)) srv.listen(5) conn, addr = srv.accept() data = conn.recv(1024) conn.sendall(data) conn.close()
Python — Client import socket cli = socket.socket( socket.AF_INET, socket.SOCK_STREAM ) cli.connect(( "10.0.14.100", 9090 )) cli.sendall(b"Hello!") resp = cli.recv(1024) print(resp) cli.close()
