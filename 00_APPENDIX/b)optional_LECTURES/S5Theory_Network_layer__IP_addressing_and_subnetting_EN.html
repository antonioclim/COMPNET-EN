The Day the Addresses Ran Out

3 February 2011 — IANA Exhaustion Day

On this date, the Internet Assigned Numbers Authority (IANA) handed the last five /8 blocks of IPv4 space to the Regional Internet Registries. After three decades of exponential growth, the pool of 4.3 billion publicly routable addresses was empty.

Yet the Internet kept growing. Billions of smartphones, IoT sensors and cloud instances came online. How?

By the end of this lecture you will understand the addressing system that both caused and survived this crisis — and you will be able to design efficient address plans yourself.

A question to hold in mind

If addresses ran out in 2011, why can you still get a public IPv4 address today? We shall return to this around slide 34.

Where We Stand: From Signals to Frames

Recall from Week 4

Last week we examined the physical layer (electrical signals, encoding) and the data link layer (Ethernet frames, MAC addresses, switches). A frame can travel between devices on the same local network — but only within a single broadcast domain.

This raises a fundamental architectural question: a MAC address can deliver a frame to any device on the same LAN segment. But what happens when the destination is on a different network, possibly on another continent?

Layer 2 has no mechanism for this. The frame simply stops at the boundary.

The Problem: Why MAC Alone Fails

PropertyMAC Address (Layer 2)What We Need for Global Reach
StructureFlat — no hierarchyHierarchical (network + host)
ScopeLocal segment onlyEnd-to-end across the Internet
AssignmentBurnt into hardware by manufacturerConfigurable, location-aware
RoutingImpossible — no aggregationAggregation enables scalable routing
Think about it: a router has millions of possible destinations. Could it store a separate entry for every MAC address on Earth?

No. With roughly 30 billion connected devices, a flat lookup table would be unmanageable. Hierarchical addressing solves this by grouping hosts into networks: a router needs only one entry per network prefix, not one per device. This is precisely what IP provides.

The Network Layer in Context

Layer 3 — the Network Layer

Responsible for logical addressing (IP addresses) and routing — determining the path packets take across interconnected networks from source to destination.

Functions and Protocol Types

Core Functions

  • Logical addressing: assigning globally meaningful addresses to interfaces
  • Routing: forwarding packets hop by hop from source to destination
  • Fragmentation (where applicable): splitting datagrams that exceed the Maximum Transmission Unit (MTU) and reassembling at the destination

Two Categories of Protocols

Routable protocols

Carry user data from upper layers. Example: IP (Internet Protocol).

Routing protocols

Help routers discover and maintain routes. Examples: OSPF, RIP, BGP. We shall study these in Week 7.

Checkpoint: Layer 2 vs Layer 3

A packet travels from Bucharest to Tokyo, passing through 12 routers. Which statement is correct?
A) Both MAC and IP addresses remain unchanged at every hop
B) Both MAC and IP addresses change at every router
C) IP addresses stay constant; MAC addresses are rewritten at each hop
D) IP addresses change at each hop; MAC addresses stay constant
✓ Correct. IP addresses identify the end-to-end conversation (source and destination hosts). MAC addresses identify the next hop on the local link. Each router strips the old Layer 2 frame, reads the IP destination, looks up the next hop, and builds a new frame with fresh source/destination MAC addresses. This is the fundamental division of labour between Layer 2 and Layer 3.
✗ Not quite. Remember: MAC addresses have local significance only — they are rewritten by every router along the path. IP addresses, by contrast, remain constant from source to destination (barring NAT, which we cover in Week 6). This hop-by-hop rewriting of Layer 2 headers is what makes inter-network routing possible.

IP: The Fundamental Protocol of the Internet

Internet Protocol (IP)

A connectionless, best-effort protocol that provides logical addressing and routing for datagrams. IP makes no guarantees about delivery, ordering or integrity — those responsibilities belong to higher layers (TCP, for instance).

Two Versions in Use

  • IPv4 (RFC 791, 1981): 32-bit addresses, very widespread
  • IPv6 (RFC 8200, 2017): 128-bit addresses, designed to replace IPv4

IPv4 Address Structure

An IPv4 address is a 32-bit number, written in dotted-decimal notation: four octets (bytes) separated by full stops, each ranging from 0 to 255.

# Example: 192.168.10.14 Decimal: 192 . 168 . 10 . 14 Binary: 11000000 10101000 00001010 00001110 <--- network portion ---><-- host -->

Every address consists of two logical parts: a network portion (identifying the subnet) and a host portion (identifying a device within that subnet). The boundary between them is defined by a prefix length or subnet mask.

Address space: a finite resource

Total IPv4 space: 232 = 4,294,967,296 addresses. With 8+ billion humans and tens of billions of devices, this is far from sufficient.

IPv4 Communication Types

Unicast

One sender → one receiver. The most common type. Each packet has a single source and a single destination IP.

Example: your browser requesting a web page.

Multicast

One sender → a group of receivers who have joined a multicast group address (224.0.0.0/4).

Example: IPTV, live stock-market feeds.

Broadcast

One sender → all hosts on the local subnet. The destination is set to the subnet broadcast address (all host bits = 1).

Example: ARP requests, DHCP discovery.

Key distinction

Routers do not forward broadcasts beyond the local subnet. This is by design: it limits broadcast storms and keeps traffic contained. Multicast, by contrast, can be routed if the infrastructure supports it.

Predict: What Happens at the Boundary?

Scenario

Host A on subnet 192.168.1.0/24 sends a broadcast packet (destination 192.168.1.255). Router R1 connects this subnet to subnet 10.0.0.0/24. Does Host B on the 10.0.0.0 subnet receive the broadcast?

Think for 30 seconds, then reveal the answer.

No. Routers define broadcast domain boundaries. The broadcast with destination 192.168.1.255 is confined to the 192.168.1.0/24 subnet. R1 will not forward it to the 10.0.0.0/24 segment. This containment is one of the primary reasons we use subnetting — to control broadcast traffic.

If Host A needs to reach Host B, it must use Host B's unicast IP address, and R1 will route the packet between subnets.

Key Fields: TTL and Protocol

TTL (Time To Live)

Decremented by 1 at each router. When it reaches 0, the packet is discarded and an ICMP "Time Exceeded" message is sent back to the source. This prevents packets from looping indefinitely through misconfigured networks.

What everyday tool exploits TTL expiry to map the network path?

traceroute (Linux/macOS) or tracert (Windows). It sends packets with TTL=1, then 2, then 3… Each router along the path responds with an ICMP error, revealing its IP address.

Indicates which transport-layer protocol is encapsulated in the payload:

Historical Address Classes (Deprecated)

In practice, classful addressing is no longer used. The rigid boundaries between /8, /16 and /24 caused massive waste: a mid-sized company needing 1,000 hosts would receive a Class B (/16) with 65,534 addresses — wasting over 98% of the allocation.

Why Classes Failed: The Allocation Trap

Scenario: a university with 500 workstations

Under classful rules, this university cannot use a Class C (maximum 254 hosts). It must request a Class B, receiving 65,534 addresses. That wastes 65,034 addresses — locked away, unusable by anyone else.

Multiply this waste across thousands of organisations. What happens?

The address space depletes far faster than the actual number of connected devices would require. By 1993 the problem was acute, leading to CIDR (Classless Inter-Domain Routing, RFC 1519) — a system that allows networks of any size, not just the three rigid classes.

CIDR replaced the class system with a simple, flexible notation: /n, where n can be any value from 0 to 32.

CIDR: Classless Inter-Domain Routing

CIDR (RFC 4632)

A method of allocating and notating IP addresses that abandons fixed class boundaries. Networks of any size can be defined by specifying a variable-length prefix.

Notation

address/prefix_length

  • 192.168.10.0/24 — 256 addresses
  • 10.0.0.0/8 — 16.7 million addresses
  • 172.16.50.0/26 — 64 addresses

What the Prefix Means

The number after the slash indicates how many leading bits identify the network portion. The remaining bits identify hosts within that network.

A /24 means 24 network bits and 8 host bits. A /26 means 26 network bits and only 6 host bits.

Prefix Length and Subnet Mask

The prefix length corresponds directly to a subnet mask: a 32-bit value with contiguous 1s followed by 0s.

# /24 prefix Prefix: /24 (24 network bits, 8 host bits) Mask: 11111111.11111111.11111111.00000000 Decimal: 255.255.255.0 # /26 prefix Prefix: /26 (26 network bits, 6 host bits) Mask: 11111111.11111111.11111111.11000000 Decimal: 255.255.255.192

The AND Operation

Apply a bitwise AND between an IP address and the subnet mask to extract the network address. The inverse (wildcard mask) extracts the host portion.

Calculating Network Parameters

Given 192.168.10.14/26, we compute every parameter systematically:

ParameterMethodResult
Prefix lengthGiven/26
Subnet mask26 bits set to 1255.255.255.192

Why 2n − 2 Usable Hosts?

Two addresses in every subnet are reserved and cannot be assigned to host interfaces.

Network Address (host bits all 0)

Identifies the subnet itself. Used in routing tables to refer to the entire network.

192.168.10.0 for /26

Broadcast Address (host bits all 1)

A packet sent to this address is delivered to every host in the subnet.

192.168.10.63 for /26

Exceptions: /31 and /32

For point-to-point links, /31 (RFC 3021) permits 2 usable addresses with no broadcast. A /32 identifies exactly one host (a loopback or host route).

Checkpoint: Usable Hosts

How many usable host addresses exist in the network 10.0.0.0/28?
A) 16
B) 14
C) 15
D) 30
✓ Correct. /28 leaves 32 − 28 = 4 host bits. Total addresses = 24 = 16. Subtract 2 for the network and broadcast addresses: 16 − 2 = 14 usable hosts. This “minus two” rule is one of the most frequently tested fundamentals in networking examinations.
✗ Not quite. A common mistake is forgetting the “minus two” rule. For /28: host bits = 4, so total addresses = 24 = 16. But the first address (.0) is the network address and the last (.15) is the broadcast — neither can be assigned to a host. Therefore 14 usable hosts. Answer A (16) confuses total addresses with usable hosts.

Why Private Addresses Matter

Private ranges are not routed on the public Internet. They allow organisations to build arbitrarily large internal networks without consuming public address space. NAT (Week 6) bridges private and public domains.

Subnetting: Why Divide a Network?

Scenario: one flat network is not enough

Your organisation has a single /24 (254 hosts). Engineering, Sales, Management and the server room all share the same broadcast domain. Every ARP request reaches every device. A misconfigured machine can broadcast-storm the entire network. A security breach in one department exposes all others.

Subnetting

The practice of dividing a single network into smaller, logically distinct subnets. Accomplished by “borrowing” bits from the host portion to extend the network prefix.

Traffic isolation

Smaller broadcast domains

Security

Departments separated by router ACLs

Efficiency

Allocate only what each group needs

FLSM: Fixed-Length Subnet Masking

All subnets receive an equal number of host addresses.

FLSM Calculation

To split into n equal subnets: borrow ⌈log2(n)⌉ bits. New prefix = old prefix + borrowed bits. Each subnet has 2(32 − new prefix) − 2 usable hosts.

FLSM Example: Splitting /24 into 4 Subnets

import ipaddress # Define the base network network = ipaddress.ip_network( "192.168.100.0/24" ) # Split into 4 subnets (+2 bits) subnets = list( network.subnets(prefixlen_diff=2) ) for i, sn in enumerate(subnets, start=1): hosts = list(sn.hosts()) print( f"Subnet {i}: {sn} " f"Range: {hosts[0]}-{hosts[-1]}" )

VLSM: Variable-Length Subnet Masking

VLSM

A subnetting technique that allocates subnets of different sizes according to actual requirements. Larger groups receive larger blocks; smaller groups receive smaller ones. This eliminates the waste inherent in FLSM.

Before we formalise: if you need subnets for 60, 20 and 2 hosts from a /24, what prefixes would you choose?

60 hosts → need 62 usable → /26 (26 − 2 = 62). 20 hosts → need 30 usable → /27 (25 − 2 = 30). 2 hosts → /30 (22 − 2 = 2). Always choose the smallest prefix that satisfies the requirement.

VLSM Example: 60, 20, 10 and 2 Hosts

Given 172.16.0.0/24, allocate for: 60, 20, 10 and 2 hosts (sorted largest first).

StepRequirementPrefix NeededAllocated SubnetUsable
160 hosts/26 (62 ≥ 60)172.16.0.0/2662
220 hosts/27 (30 ≥ 20)172.16.0.64/2730
310 hosts/28 (14 ≥ 10)172.16.0.96/2814
42 hosts/30 (2)172.16.0.112/302

Total consumed: 64 + 32 + 16 + 4 = 116 addresses. Remaining: 140 addresses for future growth.

The golden rule of VLSM

Always allocate the largest subnet first. This prevents alignment issues: each subnet must start on a boundary that is a multiple of its block size.

Checkpoint: VLSM Prefix Selection

You need a subnet for exactly 25 hosts. Which is the smallest prefix that accommodates this requirement?
A) /26 (62 usable hosts)
B) /27 (30 usable hosts)
C) /28 (14 usable hosts)
D) /25 (126 usable hosts)
✓ Correct. /28 provides only 14 hosts — insufficient. /27 provides 30 — the smallest prefix where 30 ≥ 25. /26 would work but wastes 37 addresses. In VLSM, always pick the tightest fit: /27.
✗ Reconsider. The principle is: choose the smallest block (longest prefix) where 2host bits − 2 ≥ requirement. /28 gives 14 (too few). /27 gives 30 (sufficient). /26 gives 62 (works but wastes space). The correct answer is /27.

FLSM vs VLSM and Common Mistakes

Five common subnetting errors

1. Forgetting the −2 rule (allocating /28 for 16 hosts). 2. Overlapping subnets (verify: broadcast < next network). 3. Wrong gateway (gateway must be in the same subnet as the host). 4. IP forwarding disabled on the router (sysctl -w net.ipv4.ip_forward=1). 5. Assigning the network or broadcast address to a host interface.

Returning to the Hook: IPv6 as the Long-Term Answer

Remember slide 2?

We asked: if IANA ran out of IPv4 addresses in 2011, why can you still obtain a public IPv4 address today? The short answer: NAT (Week 6), aggressive reclamation of unused blocks and a secondary market for IPv4 space have stretched the supply. But these are stopgaps. The permanent solution is IPv6 — with an address space so vast that exhaustion is, for practical purposes, impossible.

Address Space Scale

IPv4: 232 ≈ 4.3 × 109

IPv6: 2128 ≈ 3.4 × 1038

Enough to assign ~5 × 1028 addresses to every person alive.

Key IPv6 Improvements

  • Simplified, fixed-size header (40 bytes)
  • No broadcast — multicast replaces it
  • Stateless autoconfiguration (SLAAC)
  • Restores true end-to-end connectivity

IPv6 Header Format (RFC 8200)

The IPv6 header is a fixed 40 bytes — no variable-length options, no header checksum (that burden is pushed to upper layers and the link layer).

Version64 bits
Traffic ClassQoS8 bits
Flow LabelPer-flow ID20 bits
Payload LengthBytes after header16 bits
Next HeaderTCP=68 bits
Hop Limit1–2558 bits
Source Address128 bits 
Destination Address128 bits 

Next Header serves double duty: it identifies the transport protocol and chains extension headers (Hop-by-Hop, Routing, Fragment, etc.) when present.

IPv6 Address Representation

128 bits as eight 16-bit hexadecimal groups separated by colons. Two shortening rules apply:

  1. Strip leading zeros within each group: 0db8 → db8
  2. Compress the longest run of consecutive all-zero groups with :: (once only)
# Full form 2001:0db8:0000:0000:0000:0000:0000:0001 # Remove leading zeros 2001:db8:0:0:0:0:0:1 # Compress zero run with :: 2001:db8::1
Challenge: shorten fe80:0000:0000:0000:00ff:0000:0000:1234

fe80::ff:0:0:1234. The :: replaces the longest zero run (groups 2–4). The remaining 0000 groups become 0. Note: 00ff becomes ff (leading zeros stripped).

IPv6 in URLs

Because colons conflict with the port separator, IPv6 addresses in URLs must be enclosed in square brackets: http://[2001:db8::1]:8080

IPv6 Communication Types and Scopes

Communication Types

Unicast

One sender → one receiver.

Multicast

One sender → a group. Replaces IPv4 broadcast.

Anycast

The same address is assigned to multiple nodes; the packet is delivered to the nearest one (determined by routing). Used in DNS root servers and CDNs.

Address Scopes

  • Link-local: valid on a single link only (fe80::)
  • Unique local: valid within an organisation (like RFC 1918)
  • Global: globally routable (2000::/3)
  • Multicast scope: encoded in the address itself (interface-local, link-local, site-local, global)

Notice: there is no broadcast in IPv6. The all-nodes multicast address ff02::1 achieves the equivalent effect on a link.

IPv6 Subnetting in Practice

IPv6 subnetting is far simpler than IPv4. The standard model allocates a /48 prefix to an organisation and uses /64 for every LAN.

# Standard allocation model [ 48-bit global prefix ] [ 16-bit subnet ID ] [ 64-bit interface ID ] # Example: organisation receives 2001:db8:10::/48 Subnet 1: 2001:db8:10:0001::/64 # servers Subnet 2: 2001:db8:10:0002::/64 # LAN clients Subnet 3: 2001:db8:10:0003::/64 # IoT devices Subnet 4: 2001:db8:10:00ff::/64 # transit # Number of /64 subnets in a /48: 2^(64-48) = 65,536

Why always /64 for LANs?

SLAAC (Stateless Address Autoconfiguration) requires a /64 prefix to function. Equipment and protocols universally expect this boundary. Deviating from /64 on a LAN will break autoconfiguration and reduce interoperability.

Checkpoint: IPv6 Fundamentals

Which statement about IPv6 is incorrect?
A) IPv6 replaces broadcast with multicast
B) The IPv6 header is a fixed 40 bytes
C) IPv6 routers may fragment packets in transit
D) Link-local addresses (fe80::) are auto-configured
✓ Correct. In IPv6, only the source host may fragment packets; routers along the path never fragment. If a packet is too large for a link, the router sends an ICMPv6 “Packet Too Big” message, and the source must reduce the packet size. This design simplifies router processing and is one of the key differences from IPv4.
✗ The incorrect statement is C. IPv6 moved fragmentation responsibility entirely to the source host. Routers do not fragment IPv6 packets — instead, they return an ICMPv6 “Packet Too Big” message. This is by design: it simplifies router hardware and encourages path MTU discovery.

Final Challenge: Synthesis

An organisation requires subnets for 100, 50, 25 and 10 hosts. Using VLSM from 192.168.0.0/24, which prefix should be assigned to the 100-host subnet?
A) /25 (126 usable hosts)
B) /26 (62 usable hosts)
C) /24 (254 usable hosts)
D) /27 (30 usable hosts)
✓ Correct. 100 hosts requires 2n − 2 ≥ 100, so n = 7 (27 − 2 = 126). That means 32 − 7 = /25. A /26 only provides 62 — insufficient. A /24 wastes 154 addresses and leaves no room for the remaining subnets. This question tests the full VLSM reasoning chain: requirement → host bits → prefix → allocation.
✗ Remember the method: find the smallest n such that 2n − 2 ≥ 100. That gives n = 7, so the prefix is 32 − 7 = /25 (126 usable hosts). /26 gives only 62 — too few. /24 gives 254 — it works, but wastes most of the /24 on a single subnet, leaving nothing for the remaining three groups.
