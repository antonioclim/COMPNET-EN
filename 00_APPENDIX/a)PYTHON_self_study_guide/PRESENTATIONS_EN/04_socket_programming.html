🔌 What is a Socket?

A socket is a bidirectional communication endpoint. Think of it as a "plug" through which your application connects to the network.

Socket = (IP, Port, Protocol)

Server
192.168.1.10:8080
TCP Socket
⟷
Client
192.168.1.20:54321
TCP Socket

A unique connection = (IP_src, Port_src, IP_dst, Port_dst, Protocol)

📍 Components of a Socket

IP Address: Identifies the machine on the network

Port: Identifies the application on that machine (0-65535)

Protocol: TCP or UDP

🎯 Common Ports

80 - HTTP, 443 - HTTPS

22 - SSH, 53 - DNS

8080, 9090 - Development

📦 TCP vs UDP — Choosing the Protocol

🔵 TCP (Transmission Control Protocol)

Characteristics:

  • Connection-oriented (requires handshake)
  • Guarantees in-order delivery
  • Retransmits lost packets
  • Flow control and congestion control

Used for: HTTP, SSH, Email, FTP

Socket type: SOCK_STREAM

🟠 UDP (User Datagram Protocol)

Characteristics:

  • Connectionless (fire and forget)
  • Does not guarantee delivery
  • Does not guarantee ordering
  • Minimal overhead, low latency

Used for: DNS, Gaming, Streaming, VoIP

Socket type: SOCK_DGRAM

🔑 Simple Rule

TCP: When you need reliability (data that MUST NOT be lost)

UDP: When you need speed and can tolerate losses (streaming, gaming)

🖥️ TCP Server Flow

socket()
↓
bind(ip, port)
↓
listen(backlog)
↓
accept() → conn
↓
recv() / send()
↓
close()

1. socket()

Creates the socket (the endpoint)

2. bind(ip, port)

Associates the socket with an address and port

3. listen(backlog)

Marks the socket as passive (listening for connections)

4. accept()

Blocks until a client connects; returns the connection

5. recv() / send()

Bidirectional communication over the connection

💻 TCP Server — Complete Code

🐍 server_tcp.py (from 02roWSL/src/exercises/) 
import socket

# Constants
HOST = "127.0.0.1"   # Localhost
PORT = 9090          # Non-privileged port

# Context manager: socket closes automatically!
with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
    # Allow address reuse (avoids "Address already in use")
    server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)

    # Step 1: Bind to address and port
    server.bind((HOST, PORT))

    # Step 2: Start listening (max 5 pending connections)
    server.listen(5)
    print(f"🖥️  TCP Server listening on {HOST}:{PORT}")

    # Step 3: Accept connections (infinite loop)
    while True:
        conn, addr = server.accept()  # Blocks here
        with conn:
            print(f"✅ Connection from {addr}")

            # Receive data
            data = conn.recv(1024)
            if not data:
                break

            print(f"📩 Received: {data.decode('utf-8')}")

            # Send response
            response = f"Server received: {data.decode('utf-8')}"
            conn.sendall(response.encode('utf-8'))
Server Output

✅ Connection from ('127.0.0.1', 54321)
📩 Received: Hello Server!
✅ Connection from ('127.0.0.1', 54322)
📩 Received: Test message

⚠️ SO_REUSEADDR

Without this option, if you stop and restart the server quickly, you will get the error "Address already in use". The socket remains in TIME_WAIT until the kernel releases it.

💻 TCP Client — Complete Code

🐍 client_tcp.py 
import socket

HOST = "127.0.0.1"
PORT = 9090

# Context manager handles automatic closing
with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as client:
    # Connect to server
    client.connect((HOST, PORT))
    print(f"🔗 Connected to {HOST}:{PORT}")

    # Send message
    message = "Hello Server!"
    client.sendall(message.encode('utf-8'))
    print(f"📤 Sent: {message}")

    # Wait for response
    response = client.recv(1024)
    print(f"📥 Response: {response.decode('utf-8')}")

print("👋 Connection closed")
Client Output
🔗 Connected to 127.0.0.1:9090
📤 Sent: Hello Server!
📥 Response: Server received: Hello Server!
👋 Connection closed

send() vs sendall()

send() may send fewer bytes than requested

sendall() guarantees complete delivery

→ Use sendall() for safety!

recv(1024)

1024 = maximum number of bytes to read

May return fewer bytes if less data is available

→ Always check the length of the received data!

🟠 UDP — Connectionless

UDP has no connection — you send datagrams directly, without a handshake.

UDP Server

import socket

with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
    s.bind(("127.0.0.1", 9091))
    print("🟠 UDP Server on :9091")

    while True:
        # recvfrom also returns the sender address
        data, addr = s.recvfrom(1024)
        print(f"From {addr}: {data.decode()}")

        # Reply to client
        s.sendto(b"ACK", addr)

UDP Client

import socket

with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
    # No connect() — send directly!
    server = ("127.0.0.1", 9091)

    s.sendto(b"Hello UDP!", server)
    print("📤 Datagram sent")

    # Optional: receive a response
    data, addr = s.recvfrom(1024)
    print(f"📥 Response: {data}")

🔑 Key Differences: TCP vs UDP

TCP:SOCK_STREAM, connect(), send()/recv()
UDP:SOCK_DGRAM, sendto(addr)/recvfrom()

🔒 Context Managers — The with Statement

Context managers guarantee that resources are released correctly, even if an exception occurs.

❌ Without with (risk of leak)

# If an exception occurs before close(),
# the socket stays open!

sock = socket.socket(...)
sock.connect((host, port))
sock.send(data)           # Exception here?
response = sock.recv(1024)
sock.close()              # Never reached!

✅ With with (safe)

# Socket closes AUTOMATICALLY
# upon exiting the block, guaranteed!

with socket.socket(...) as sock:
    sock.connect((host, port))
    sock.send(data)
    response = sock.recv(1024)
# Automatic close happens here

💡 Golden Rule

ALWAYS use with for sockets (and files)! It is the Python equivalent of try-finally in Java or RAII in C++.

🐍 Nested context managers (server) 
# Both sockets close automatically
with socket.socket(...) as server:
    server.bind((HOST, PORT))
    server.listen()
    conn, addr = server.accept()
    with conn:  # Second context manager for the connection
        data = conn.recv(1024)
        conn.sendall(response)

What is the correct order of calls for a TCP server?
A socket → listen → bind → accept
B socket → bind → listen → accept
C socket → accept → bind → listen

Hint 1/3

You must first bind the socket to an address before you can listen on it.

Hint 2/3

listen() marks the socket as passive. accept() comes after listen().

Hint 3/3

connect() is for the CLIENT, not the server. The server does bind → listen → accept.

🔧 Common Patterns in Exercises

🐍 Pattern: Echo Server (returns whatever it receives) 
def echo_server(host: str, port: int) -> None:
    """Server that returns exactly what it receives."""
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        s.bind((host, port))
        s.listen()

        while True:
            conn, addr = s.accept()
            with conn:
                while True:
                    data = conn.recv(1024)
                    if not data:
                        break
                    conn.sendall(data)  # Echo: send back
🐍 Pattern: Client with Timeout 
def client_with_timeout(host: str, port: int, timeout: float = 5.0):
    """Client that does not block indefinitely."""
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.settimeout(timeout)  # Set timeout
        try:
            s.connect((host, port))
            s.sendall(b"Hello")
            response = s.recv(1024)
            return response
        except socket.timeout:
            print("⏱️ Timeout — the server is not responding")
            return None

⚠️ Error Handling in Socket Programming

🐍 error_handling.py 
import socket
import sys

def safe_connect(host: str, port: int) -> socket.socket | None:
    """Attempt to connect with comprehensive error handling."""
    try:
        sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        sock.settimeout(10.0)
        sock.connect((host, port))
        return sock

    except socket.timeout:
        print(f"❌ Timeout: {host}:{port} is not responding")

    except socket.gaierror as e:
        print(f"❌ DNS Error: Cannot resolve '{host}'")

    except ConnectionRefusedError:
        print(f"❌ Connection refused: Port {port} is closed")

    except OSError as e:
        print(f"❌ Network error: {e}")

    return None

🔴 Common Exceptions

ConnectionRefusedError — Port is closed or blocked by firewall

socket.timeout — Server did not respond in time

socket.gaierror — DNS cannot resolve the hostname

BrokenPipeError — Connection was interrupted

⚠️ Important Tip

ALWAYS set a timeout for network operations! Without a timeout, recv() and connect() can block your program indefinitely.
