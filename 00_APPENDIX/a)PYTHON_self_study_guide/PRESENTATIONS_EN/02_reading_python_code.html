🏷️ Dataclasses — Elegant Data Structures

Python uses @dataclass to create structures similar to struct in C. The decorator automatically generates __init__, __repr__ and other methods.

Python with @dataclass

🐍 Python 
from dataclasses import dataclass
from typing import Optional

@dataclass
class PingResult:
    """The result of a single ping."""
    sequence: int
    rtt_ms: Optional[float]
    succeeded: bool
    message: str = ""  # default value

# Usage
r = PingResult(
    sequence=1,
    rtt_ms=0.42,
    succeeded=True
)
print(r.sequence)  # 1
print(r)  # PingResult(sequence=1, ...)

Equivalent in C

📄 C 
typedef struct {
    int sequence;
    float rtt_ms;     // cannot be NULL natively
    bool succeeded;
    char message[256];
} PingResult;

// Usage
PingResult r;
r.sequence = 1;
r.rtt_ms = 0.42;
r.succeeded = true;
strcpy(r.message, "");

printf("%d", r.sequence);  // 1
// No automatic print!

🔑 Key Concepts

Optional[float] = the value can be either float or None (equivalent to nullable in other languages)

@dataclass = a decorator that automatically adds a constructor, __repr__ and comparisons

message: str = "" = a parameter with a default value (does not need to be specified at creation)

📐 Type Hints — Documenting Types

Python is dynamically typed (it does not check types at compile time) but it allows type annotations for documentation and static verification tools.

🐍 type_hints_example.py 
# Function without type hints (hard to understand)
def process(data, count):
    return data * count

# Function WITH type hints (clear!)
def process_clear(data: str, count: int = 1) -> str:
    """Repeats the string 'count' times."""
    return data * count

# Complex types
from typing import List, Dict, Optional, Tuple

def analyse_network(
    ip_addresses: List[str],                    # List of strings
    config: Dict[str, int],               # Dictionaryy str→int
    timeout: Optional[float] = None       # Float or None
) -> Tuple[int, List[str]]:              # Returns a tuple
    """Analyses a list of IP addresses."""
    pass

# Test
result = process_clear("test", 3)
print(f"Result: {result}")
Output
Result: testtesttest

⚠️ Important

Type hints serve only as documentation — Python does NOT enforce them at runtime. You can pass any type and the code will still run. Tools such as mypy perform static checking.

📝 Quiz: Check Your Understanding

🧠

Quiz 1: Dataclasses

Question 1 of 3

What does the @dataclass decorator do in Python?
A Transforms the class into a function
B Automatically generates __init__, __repr__ and other methods for the data class
C Checks types at runtime and raises errors
D Automatically converts the class to JSON format
Hint 1/3

Think about what you normally write when creating a class. What methods must you implement manually to create instances and display them?

Hint 2/3

In C, when you use a struct, you must manually initialise each field. @dataclass eliminates this requirement in Python.

Hint 3/3

@dataclass = "data class" = a class designed for holding data. It automatically generates a constructor (__init__), a text representation (__repr__) and equality comparisons (__eq__).

🧠

Quiz 2: Type Hints

Question 2 of 3

What does Optional[float] mean in the context of type hints?
A The value must be a float
B The parameter is optional and can be omitted
C The value can be either float or None
D The value will be automatically converted to a float
Hint 1/3

In the networking context, why would rtt_ms (Round-Trip Time) be optional? What happens when a packet does not arrive?

Hint 2/3

"Optional" in type hints does not mean the parameter can be omitted. It refers to the type of the value. Optional[X] = Union[X, None]

Hint 3/3

If the ping fails, there is no valid RTT. Instead of using a magic value (-1), we use None to indicate "no value".

Quiz 3: Python Syntax

Question 3 of 3

What is the Python equivalent of for (int i = 0; i < 10; i++) from C?
A for (i = 0; i < 10; i++):
B for i = 0 to 10:
C for i in range(10):
D foreach i in [0..10]:
Hint 1/3

Python uses the keyword construct for ... in ... for all iteration. It has no syntax with parentheses and semicolons.

Hint 2/3

range(n) generates numbers from 0 to n-1. Thus range(10) produces 0, 1, 2, ..., 9

The code block is delimited by indentation (spaces) rather than curly braces. The for line ends with :

🐍 sandbox_dataclass.py 
from dataclasses import dataclass
from typing import Optional

@dataclass
class PacketInfo:
    """Information about a network packet."""
    source_ip: str
    dest_ip: str
    protocol: str
    size: int
    ttl: Optional[int] = None

# Create a TCP packet
package_tcp = PacketInfo(
    source_ip="192.168.1.100",
    dest_ip="8.8.8.8",
    protocol="TCP",
    size=1500,
    ttl=64
)

# Create a UDP packet without specifying TTL
package_udp = PacketInfo(
    source_ip="192.168.1.100",
    dest_ip="1.1.1.1",
    protocol="UDP",
    size=512
)

print("TCP Packet:")
print(f"  From: {package_tcp.source_ip}")
print(f"  To: {package_tcp.dest_ip}")
print(f"  TTL: {package_tcp.ttl}")

print(f"\nUDP Packet (TTL uninitialised): {package_udp.ttl}")
print(f"\nFull representation:\n{package_tcp}")
